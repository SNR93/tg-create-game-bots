# TG Game Bot Constructor

Браузерный визуальный редактор для создания интерактивных нарративных Telegram-ботов — квесты, игры, разветвлённые истории — без единой строки кода.

---

## Что это и зачем

Большинство конструкторов Telegram-ботов заточены под простые воронки или FAQ-боты. Этот инструмент создан для **нарративных сценариев**: разветвлённые истории, инвентарь, отношения с персонажами, достижения, монетизация через Telegram Stars.

Вы рисуете граф — бот его проигрывает.

Редактор работает в браузере. Движок исполняет сценарии напрямую внутри Telegram. Всё разворачивается у вас — ваши данные, ваш сервер, ваши правила.

## Сильные стороны

### Визуальный нод-граф
Перетаскивайте, соединяйте и настраивайте ноды на бесконечном холсте на базе React Flow. Никакого YAML, JSON или кода.

### Богатая библиотека нод (35+ типов)

| Категория | Ноды |
|---|---|
| Точки входа | Глобальное меню (/start), Настройки (/settings), Свои команды, Вызвать команду, Продолжить историю |
| Сообщения | Сообщение, Цепочка сообщений, Медиа-подборка, Ввод текста, Изменить сообщение, Опрос/тест, Стикер, Геолокация |
| Логика | Проверка условий, Расчёт чисел, Случайный выбор, Задержка, Проверка подписки, HTTP-запрос, Цикл, Выход из цикла, Глобальные переменные |
| Прогресс | Переменные, Инвентарь, Показ инвентаря, Отношения, Достижения, Показ достижений, Контрольная точка |
| Монетизация | Покупка Telegram Stars, Промокод |
| Структура | Подсценарий, Возврат, Комментарий, Группа |

### Авторизация и рабочие инструменты редактора
Веб-интерфейс защищён страницей входа. В редакторе есть поиск по схеме, лор проекта, сворачивание/разворачивание нод (вертикальное), встроенная проверка сценария, автосохранение, история изменений сценария (undo/redo), история изменений каждой ноды (кто, когда, комментарий, сравнение, восстановление), копирование/вставка нод, справка с поиском и настройки с экспортом JSON и админ-панелью.

### Двунаправленные соединения
Каждая нода с единственным выходом «Продолжить» имеет **два синих выхода** — правый и левый. Это позволяет строить схемы в любом направлении: слева направо, справа налево, по кругу. Оба выхода взаимоисключающие: если один занят, второй блокируется.

### История изменений нод
При каждом сохранении (Ctrl+S) система автоматически фиксирует изменения каждой ноды: автор, время по МСК, снапшот данных. В инспекторе кнопка 🕐 открывает историю — можно добавить комментарий, сравнить с текущей версией (показывает diff по полям), восстановить старую версию или удалить запись. Ручные снапшоты с произвольным комментарием сохраняются в любой момент.

### Лор и форматирование текста
Лор хранится вместе с ботом и помогает вести персонажей, правила мира и заметки автора. Гиперссылки в лоре подсвечиваются синим цветом прямо в редакторе. Тексты сообщений поддерживают плейсхолдеры `{{Переменная}}` и Telegram HTML-разметку: жирный, курсив, подчёркивание, зачёркивание, спойлер, код и ссылки.

### Встроенный симулятор
Тестируйте полный сценарий прямо в браузере — с имитацией чата Telegram, панелью переменных, журналом отладки и таймерами задержек. Реальный бот для теста не нужен.

### Панель администратора
Просмотр и редактирование переменных, инвентаря, отношений и достижений каждого игрока в реальном времени. Поиск, сброс, удаление игроков, журнал выборов.

### Версии и резервные копии
Делайте снимок сценария в любой момент, публикуйте с постепенным выкатом (например, 10% игроков получают v2), мгновенно откатывайтесь из бэкапа. Telegram-бэкап отправляет архив со сценариями, медиа и SQL-дампом PostgreSQL (`db/tgbot.sql`); при восстановлении новые архивы восстанавливают и БД, старые архивы без дампа — только файлы.

### Аналитика
Воронка конверсии, тепловая карта посещений нод, разбивка выборов игроков — всё в панели администратора.

### Монетизация через Telegram Stars
Создавайте товары в панели, размещайте ноду «Покупка» в любом месте истории. Платёж обрабатывает Telegram — никаких платёжных провайдеров.

### Redis-сессии
Сессии игроков переживают рестарт бэкенда. Состояние сохраняется в Redis и прозрачно восстанавливается.

### Rate limiter для Telegram API
Встроенный токен-бакет (25 запросов/с) защищает от превышения лимитов Telegram при рассылках и пиковой нагрузке.

### Медиа через nginx
Загруженные файлы (фото, видео, голосовые) раздаёт nginx напрямую, минуя Node.js — с 30-дневным кешированием.

### Self-hosted, Docker-первый
Одна команда `docker compose up`. Никаких внешних сервисов, никакой привязки к облаку.

## Быстрый старт

### Требования
- Docker и Docker Compose (v2+)
- Свободные порты `4000` и `4001`

### 1. Клонировать

```bash
git clone https://github.com/SNR93/tg-create-game-bots.git
cd tg-create-game-bots
```

### 2. Запустить

```bash
docker compose up -d --build
```

Поднимаются четыре контейнера: PostgreSQL, Redis, бэкенд API, nginx-фронтенд.

### 3. Открыть

```
http://localhost:4000
```

### 4. Войти

Панель защищена авторизацией. Логины и пароли не хранятся в репозитории: задайте их через переменную окружения `AUTH_USERS` в формате `login:password,second:password`.

Пример локального `.env`:

```env
AUTH_USERS=admin:change-me
AUTH_SECRET=replace-with-long-random-secret
```

Сессия хранится 30 дней. Для стабильных сессий после перезапуска задайте постоянный `AUTH_SECRET`.

### 5. Создать бота

1. Нажмите **+ Новый бот**
2. Постройте сценарий в редакторе нод
3. Сохраните — **Ctrl+S**

### 6. Подключить к Telegram

1. Получите токен у [@BotFather](https://t.me/BotFather) командой `/newbot`
2. Откройте панель **Telegram** в редакторе
3. Вставьте токен и нажмите **Запустить**

Бот начнёт работать немедленно. Напишите `/start` в Telegram, чтобы проверить.

## Webhook-режим (для продакшена)

Для развёртывания с публичным доменом установите `PUBLIC_BASE_URL` перед запуском:

```bash
PUBLIC_BASE_URL=https://ваш-домен.ru docker compose up -d
```

Бэкенд зарегистрирует Telegram-вебхук вместо long polling.

## Возможности редактора

- **Схема:** перетаскивание нод, группы, комментарии, пунктирные связи-комментарии, удаление, дублирование, копирование/вставка, миникарта и масштаб.
- **Поиск:** находит ноды по названию, текстам, командам и настройкам; найденные блоки подсвечиваются на холсте.
- **Лор:** отдельное окно для правил мира, персонажей и авторских заметок; сохраняется в JSON бота.
- **Клавиатура:** кнопки поддерживают несколько условий показа (AND), подсказки по выбранному источнику и системные Telegram-плейсхолдеры при вводе `telegram.` или `{{telegram.`. В редакторе кнопки с условиями отмечаются синей шестерёнкой; игрок её не видит.
- **Проверка:** валидирует сценарий перед запуском: незаполненные поля, лимиты Telegram, проблемные связи и медиа.
- **Симулятор:** проигрывает сценарий в браузере, показывает состояние переменных, инвентаря, отношений, достижений и журнал отладки.
- **Telegram:** запуск/остановка бота, просмотр статуса и логов. Без `PUBLIC_BASE_URL` используется long polling, с публичным HTTPS-адресом — webhook.
- **Админ-панель:** игроки, переменные, инвентарь, отношения, достижения, промокоды, товары Stars, версии, бэкапы, аналитика и рассылки. Telegram-бэкап включает сценарии, медиа и PostgreSQL-дамп для восстановления прогресса игроков и служебных данных.
