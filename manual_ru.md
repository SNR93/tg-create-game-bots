## 1. Быстрый старт

### Требования

- **Docker** и **Docker Compose** (v2+)
- Свободные порты: `4000` (фронтенд), `4001` (API)
- Интернет-соединение на сервере (для связи с Telegram API)

### Установка

```bash
git clone https://github.com/SNR93/tg-create-game-bots.git
cd tg-create-game-bots
docker compose up -d --build
```

Команда поднимает 4 контейнера:

| Контейнер | Роль | Порт |
|---|---|---|
| `tgbot-frontend` | nginx + React-приложение | 4000 |
| `tgbot-backend` | Node.js API + движок бота | 4001 |
| `tgbot-postgres` | База данных игроков | 5433 (локальный) |
| `tgbot-redis` | Хранилище сессий | 6380 (локальный) |

### Первый запуск

1. Откройте **http://localhost:4000**
2. Нажмите **+ Новый бот**
3. Введите название и нажмите **Создать**
4. Откроется редактор с нодой «Глобальное меню» — с неё начинается любой сценарий

### Остановка и перезапуск

```bash
docker compose down       # остановить (данные сохраняются)
docker compose up -d      # запустить снова
docker compose down -v    # остановить и удалить все данные (!)
```

## 2. Интерфейс редактора

### Панель инструментов (верхняя)

| Кнопка | Действие |
|---|---|
| Название бота | Редактировать имя (Enter для сохранения) |
| **Сохранить** / Ctrl+S | Сохранить сценарий |
| **Скачать JSON** | Выгрузить сценарий как файл |
| **Тест** | Открыть встроенный симулятор |
| **Telegram** | Подключить и запустить бота |
| **Админ** | Открыть панель управления игроками |
| **История** | Просмотр и откат изменений (undo/redo) |
| **?** | Справка по нодам |
| ← | Вернуться к списку ботов |

### Холст

- **Перемещение**: зажать пробел + тащить, или тащить за пустую область
- **Масштаб**: колесо мыши или Ctrl+/−
- **Выделить несколько нод**: зажать Ctrl и кликать, или тащить рамкой
- **Удалить ноду**: выделить → Del или Backspace
- **Дублировать**: выделить → Ctrl+D

### Панель нод (левая)

Каталог всех доступных нод, разбитых по категориям. Перетащите ноду на холст или нажмите на неё — она добавится в центр экрана.

### Инспектор (правая панель)

Появляется при клике на ноду. Настройки ноды: тексты, переменные, условия, файлы.

Кнопка **🕐** в шапке инспектора открывает **историю изменений этой ноды** (см. ниже).

### История изменений ноды

При каждом сохранении (Ctrl+S) система автоматически записывает изменения данных каждой ноды. Панель истории открывается кнопкой **🕐** в инспекторе.

| Действие | Описание |
|---|---|
| 💾 Сохранить снимок | Ручной снапшот с произвольным комментарием |
| ▼ / ▶ | Раскрыть/свернуть запись |
| ✏️ Комментарий | Редактировать метку к версии |
| 🔍 Сравнить с текущим | Показывает diff по изменённым полям |
| ↩ Восстановить | Применяет историческую версию (сохраните Ctrl+S для подтверждения) |
| 🗑 Удалить | Удалить запись из истории |

Каждая запись содержит: автор, время по **МСК**, комментарий, снапшот данных. Хранится до 50 записей на ноду — старые обрезаются автоматически.

### Связи между нодами

- Перетащите от **выхода** одной ноды к **левому входу** другой
- Большинство нод имеют один выход «Продолжить» — с **двух сторон**: правый (→) и левый (←)
- Оба выхода взаимоисключающие: если правый соединён, левый блокируется, и наоборот
- Клавиатуры и ветвления имеют **отдельный выход на каждый вариант** (тоже с обеих сторон)
- Связи-комментарии (пунктирные оранжевые) не влияют на выполнение

### Группы

Выделите несколько нод (Ctrl+клик или рамкой), нажмите правую кнопку → **Группа**. Группа объединяет ноды визуально и перемещает их вместе.

## 3. Все типы нод

### Точки входа и команды

#### Глобальное меню (`menuNode`)
**Открывается командой `/start`**. Это главная точка входа в бота — показывается каждый раз, когда игрок пишет /start, независимо от текущего места в истории.

Типичное содержимое: кнопки «Продолжить историю», «Профиль», «Магазин», «Настройки».

> Нода не перезаписывает прогресс игрока. После нажатия любой кнопки управление переходит к подключённой ноде.

---

#### Настройки (`settingsNode`)
**Открывается командой `/settings`**. Независимая ветка для пользовательских параметров: уведомления, язык, режим отображения.

---

#### Своя команда (`customCommandNode`)
Создаёт произвольную команду Telegram (например, `/profile`, `/shop`, `/map`).

**Настройки в инспекторе:**
- **Команда** — имя без `/`, только латинские буквы, цифры, `_`
- **Псевдонимы** — дополнительные имена через запятую
- **Описание** — текст в меню команд Telegram
- **Показать в меню** — отображать ли команду в системном меню Telegram

Аргументы команды доступны через плейсхолдеры `{{command.args}}`, `{{command.arg1}}`, `{{command.arg2}}` и т.д.

---

#### Вызвать команду (`invokeCommandNode`)
Принудительно открывает командную ноду прямо из сценария — как если бы игрок сам написал команду.

**Использование:** выберите целевую ноду (меню, настройки или свою команду) в выпадающем списке. Выбранная нода откроется в режиме оверлея. После нажатия «Продолжить историю» сюжет продолжится со следующего блока после этой ноды.

**Пример:** конец главы → автоматически показать меню → следующая глава.

---

#### Продолжить историю (`continueStoryNode`)
Завершает независимую ветку (меню, настройки, команда) и возвращает игрока к последней позиции в истории.

> У ноды нет выхода — переход вычисляется автоматически.

---

### Сообщения

#### Сообщение (`simpleMessageNode`)
Отправляет одно сообщение: **текст**, **фото**, **видео**, **голосовое**, **аудио** или **документ**.

**Поддерживает:**
- Плейсхолдеры `{{ИмяПеременной}}` в тексте
- Защищённый контент (спойлер для фото/видео, запрет пересылки)
- Видео-кружки (video note)

---

#### Цепочка сообщений (`messageChainNode`)
Последовательно отправляет несколько сообщений из одного блока. Каждое сообщение может содержать задержку перед отправкой.

**Когда использовать:** диалоги, кат-сцены, серии реплик с единым переходом после.

---

#### Медиа-подборка (`mediaNode`)
Отправляет один или несколько медиафайлов. Поддерживает **режим альбома** (2–10 фото или видео одним сообщением).

---

### Логика сценария

#### Выбор игрока (`keyboardNode`)
Показывает inline-кнопки Telegram и ждёт нажатия. Каждая кнопка — отдельный выход на холсте.

**Настройки:** добавляйте и удаляйте кнопки, редактируйте подписи (поддерживают плейсхолдеры).

---

#### Проверка условий (`branchingNode`)
Проверяет значения переменных и выбирает подходящую ветку. Ветки проверяются **сверху вниз**, срабатывает первая, у которой выполнены все условия.

**Ветка без условий** работает как «Иначе» — поставьте её последней.

**Операторы:** `==`, `!=`, `>`, `<`, `>=`, `<=`

---

#### Случайный выбор (`randomNode`)
Генерирует случайное целое число в заданном диапазоне и выбирает ветку, чей интервал «от—до» содержит это число.

**Настройка в инспекторе:**
- Диапазон: «Случайное число от N до M»
- Варианты задаются в одну строку: «Название | от | до»
- Интервалы должны покрывать весь диапазон без пересечений

**Пример:** диапазон `1..10`, ветка «Обычный» `1..7`, ветка «Редкий» `8..10` → 70% / 30%.

---

#### Расчёт чисел (`formulaNode`)
Вычисляет арифметическую формулу и записывает результат в числовую переменную.

**Поддерживаемые операторы:** `+`, `−`, `*`, `/`, `()`, плейсхолдеры `{{переменная}}` (только числовые).

**Проценты:** `X%` — это десятичное сокращение (`50% = 0.5`). Запись `выражение - 50%` означает «умножить на `(1 − 50/100)`», то есть уменьшить на 50% от самого себя. `выражение + 25%` — увеличить на 25%.

Пример: `(5+5)*2-50%` → `20 * 0.5` = `10`.

Нецелые результаты округляются вверх (`Math.ceil`). Инспектор показывает результат мгновенно на основе значений по умолчанию, а также список всех доступных переменных.

---

#### Задержка (`delayNode`)
Приостанавливает сценарий на заданное количество секунд через серверную очередь. Бот продолжит сценарий автоматически — даже если игрок не пишет.

---

### Прогресс и экономика

#### Переменные (`variableNode`)
Создаёт и изменяет переменные:

| Тип | Значения | Операции |
|---|---|---|
| `boolean` | true / false | set |
| `number` | любое число | set, increment, decrement |
| `text` | строка | set |

Переменные доступны во всём сценарии через плейсхолдеры `{{НазваниеПеременной}}`.

---

#### Изменить инвентарь (`inventoryNode`)
Добавляет, удаляет или устанавливает количество предметов в инвентаре игрока.

**Операции:** `add` (добавить), `remove` (удалить, минимум 0), `set` (установить)

---

#### Изменить отношения (`relationNode`)
Меняет числовой показатель отношений с персонажем или фракцией.

**Операции:** `set`, `add`, `subtract`

---

#### Выдать достижение (`achievementNode`)
Открывает достижение игроку. Повторный вызов не создаёт дубликат. Можно показать уведомление в чате.

---

#### Контрольная точка (`checkpointNode`)
Сохраняет стабильную точку возврата. Если игрок снова напишет `/start` → перейдёт к меню, а «Продолжить историю» вернёт его именно сюда.

> Используйте в начале каждой главы или после важных сцен.

---

#### Запросить промокод (`promocodeNode`)
Запрашивает у игрока ввод промокода. Промокоды создаются в **Админ-панели → Промокоды**. Команда `/promo КОД` также работает в любом месте сценария.

**Настройка в инспекторе:**
- **Текст запроса** — что написать игроку перед вводом
- **Переменные при успехе** — список пар «переменная = значение». Все указанные переменные будут установлены при вводе правильного промокода. Можно добавить несколько.

> При неверном промокоде переменные не меняются, бот идёт по обычному выходу.

---

#### Покупка Stars (`purchaseNode`)
Открывает оплату через Telegram Stars. Товар настраивается в **Админ-панели → Магазин Stars**.

---

### Структура

#### Вызвать подсценарий (`subscenarioNode`)
Переходит к переиспользуемой ветке и после ноды «Возврат» продолжает выполнение со следующего блока.

**Настройка в инспекторе:**
- **Название этого вызова** — метка для удобства (например, «Флэшбек», «Диалог с NPC»). Отображается на карточке ноды
- **Точка входа** — нода, с которой начинается подсценарий

---

#### Возврат из подсценария (`returnNode`)
Завершает вложенную ветку и возвращает выполнение к месту вызова.

**Настройка в инспекторе:** можно явно указать из какого подсценария возвращаться — если в схеме несколько подсценариев. По умолчанию — возврат из последнего активного.

---

#### Комментарий (`commentNode`)
Заметка автора на схеме. Не влияет на выполнение бота. Можно прикрепить пунктирной связью к любой ноде.

---

#### Группа (`groupNode`)
Цветная область для организации нод. Создаётся правой кнопкой по выделению. При перемещении группы ноды внутри двигаются вместе.

## 4. Переменные и плейсхолдеры

### Создание переменных

Переменные создаются нодой **«Переменные»**. Тип определяется при первом создании и должен оставаться согласованным.

### Использование в тексте

В любом текстовом поле используйте двойные фигурные скобки:

```
Привет, {{Имя}}!
У тебя {{Монеты}} монет.
Отношения с Алисой: {{Доверие_Алисы}}
```

Если переменная не найдена, плейсхолдер остаётся видимым (`{{Имя}}`) — и сразу в логах бота появится предупреждение, чтобы вы могли быстро найти ошибку.

### Встроенные переменные команд

Внутри ветки `customCommandNode` доступны:

| Плейсхолдер | Значение |
|---|---|
| `{{command.args}}` | Все аргументы после команды |
| `{{command.arg1}}` | Первый аргумент |
| `{{command.arg2}}` | Второй аргумент |
| … | … |

**Пример:** игрок пишет `/give sword` → `{{command.arg1}}` = `sword`

## 5. Подключение Telegram-бота

### Шаг 1: Создать бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot`
3. Введите имя и username (должен заканчиваться на `bot`)
4. Скопируйте токен вида `1234567890:ABCdef...`

### Шаг 2: Подключить токен в редакторе

1. Откройте редактор бота
2. Нажмите кнопку **Telegram** в верхней панели
3. Вставьте токен и нажмите **Запустить**

Бот начнёт работать в режиме **long polling** — он самостоятельно опрашивает Telegram на наличие сообщений.

### Режимы работы

| Режим | Когда используется |
|---|---|
| **Long polling** | Без `PUBLIC_BASE_URL` — для разработки и локальных машин |
| **Webhook** | При установленном `PUBLIC_BASE_URL` — для продакшена |

### Просмотр логов

В панели Telegram отображаются последние 300 событий: входящие сообщения, выборы игроков, ошибки, предупреждения о несуществующих переменных.

### Остановка бота

Нажмите **Остановить** в панели Telegram. Бот можно запустить снова в любой момент.

## 6. Тестирование в симуляторе

Симулятор позволяет протестировать сценарий прямо в браузере без подключения к Telegram.

### Открытие

Нажмите кнопку **Тест** в верхней панели редактора.

### Интерфейс

| Панель | Содержимое |
|---|---|
| **Левая** | Текущие значения переменных (можно редактировать вручную) |
| **Центральная** | Имитация чата Telegram |
| **Правая** | Журнал выполнения с указанием нод |

### Команды бота

Нажмите кнопку ☰ в поле ввода, чтобы запустить любую команду из тех, что настроены на схеме: `/start`, `/settings`, свои команды.

### Особенности симулятора

- **Задержки** отображаются таймером с кнопкой «Пропустить»
- **Покупка Stars** показывает карточку с кнопками «Оплатить (симуляция)» и «Пропустить»
- **Промокод** запрашивает ввод, но не проверяет реальные промокоды — любой текст принимается
- Переменные можно менять прямо во время выполнения для проверки разных ветвлений

## 7. Админ-панель

Открывается кнопкой **Админ** в верхней панели редактора.

### Игроки

Список всех пользователей, которые запустили бота. Поиск по имени, username или Telegram ID.

**Для каждого игрока доступно:**
- Редактирование переменных (тип и значение)
- Управление инвентарём (добавить, удалить, изменить количество)
- Просмотр и изменение отношений с персонажами
- Управление достижениями
- Журнал выборов (какие кнопки нажимал)
- Сброс прогресса
- Полное удаление

### Аналитика

- **Воронка конверсии** — от «начали игру» до «прошли сценарий»
- **Топ посещаемых нод** — горизонтальный бар-чарт по количеству визитов
- **Популярные выборы** — какие кнопки нажимают чаще всего
- **Сводка** — количество оплат, Stars заработано, реферальные игроки

### Промокоды

Создание промокодов с наградами. Награды указываются в формате JSON:

```json
{
  "variables": { "Монеты": 100 },
  "inventory": { "ключ_подвала": 1 }
}
```

**Параметры:**
- **Лимит использований** — оставьте пустым для неограниченного
- Промокод деактивируется автоматически после исчерпания лимита

### Магазин Stars

Добавление товаров для покупки через Telegram Stars:
- **Ключ** — стабильный идентификатор товара (используется в ноде «Покупка»)
- **Название** и **описание** — показываются в инвойсе Telegram
- **Цена** — в единицах Telegram Stars (минимум 1)
- **Награды** — JSON с переменными/предметами, которые выдаются после оплаты

### Роли

Управление доступом к проекту: `owner`, `editor`, `viewer`.

## 8. Версии и резервные копии

### Версии сценария

**Создать снимок**: Adminпанель → Версии → «Создать снимок версии»

Снимок фиксирует текущее состояние сценария. После создания его можно **опубликовать** с указанием **процента выката** (rollout):

```bash
cd tg-create-game-bots

```

### Настройка обратного прокси (nginx + SSL)

Пример конфигурации для Nginx Proxy Manager или ручного nginx:

```nginx
server {
    listen 443 ssl;
    server_name yourdomain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://localhost:4000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

```

Добавьте `PUBLIC_BASE_URL` в `docker-compose.yml`:


В webhook-режиме бот не опрашивает Telegram — Telegram сам шлёт обновления на сервер. Это быстрее и надёжнее для продакшена.

## 13. Часто задаваемые вопросы

### Бот не запускается, ошибка «Telegram отклонил токен»
Проверьте токен в настройках бота у @BotFather. Убедитесь, что бот не был удалён и токен не был сброшен.

### «Этот Telegram-бот уже запущен в другом месте»
Токен используется в другом процессе или экземпляре приложения. Остановите другой экземпляр или подождите несколько секунд — Telegram сам разрывает старое соединение.

### Не удалось подключиться к Telegram API
Сервер не имеет доступа к `api.telegram.org`. Проверьте интернет-соединение и файервол. В некоторых странах Telegram заблокирован — используйте VPN на сервере.

### Плейсхолдер `{{Переменная}}` отображается как есть в сообщении
Переменная не найдена в момент выполнения этой ноды. Проверьте:
- Нода **«Переменные»** для этой переменной стоит **до** ноды с текстом в сценарии
- Имя переменной совпадает (учитывайте регистр)
- В логах Telegram-панели будет предупреждение с точным именем отсутствующей переменной

### После восстановления из резервной копии ничего не изменилось
Обновите страницу браузера (Ctrl+Shift+R) после восстановления.

### Как переименовать существующую переменную?
Прямого переименования нет. Создайте новую ноду «Переменные» с новым именем, установите нужное значение, и замените все плейсхолдеры в текстах.

### Сколько игроков поддерживается?
Ограничений на количество игроков нет. Узкое место — API Telegram (30 сообщений/сек глобально), которое автоматически регулируется встроенным rate limiter'ом.

### Как запустить несколько ботов?
Создайте любое количество ботов через **+ Новый бот**. Каждый бот работает с отдельным Telegram-токеном и независимо. Все они обслуживаются одним бэкендом.

### Как посмотреть логи контейнеров?

```bash
docker logs tgbot-backend -f    # логи бэкенда в реальном времени
docker logs tgbot-frontend      # логи nginx
docker logs tgbot-postgres      # логи базы данных
```

## 14. Актуальная инструкция по всем возможностям

Этот раздел фиксирует текущую сборку редактора и дополняет старые части руководства. Если описание выше расходится с этим разделом, используйте правила из раздела 14.

### Вход и список ботов

Веб-интерфейс открывается по адресу `http://localhost:4000` и требует авторизацию.

Данные входа не хранятся в репозитории. Задайте пользователей панели через переменную окружения `AUTH_USERS`:

```env
AUTH_USERS=admin:change-me
AUTH_SECRET=replace-with-long-random-secret
```

Формат `AUTH_USERS`: `login:password,second:password`. После входа открывается список ботов. В нём можно создать нового бота, открыть существующего, увидеть автора, дату обновления и комментарий. API защищён токеном и cookie `tgbot_session`; срок сессии — 30 дней.

Для стабильных сессий после перезапуска обязательно задайте постоянный `AUTH_SECRET`.

### Верхняя панель редактора

| Элемент | Что делает |
|---|---|
| **Боты** | Возвращает к списку ботов |
| Название бота | Меняет имя текущего бота |
| **Развернуть/Свернуть ноды** | Переключает компактный вид блоков на схеме |
| **Лор** | Открывает заметки мира, персонажей и правил проекта |
| **Поиск** | Ищет по нодам, текстам, командам, ключам и настройкам |
| **Тест** | Запускает встроенный симулятор |
| **Проверить** | Проверяет сценарий на ошибки, лимиты Telegram и незаполненные поля |
| **Telegram** | Запуск, остановка, статус и логи Telegram-бота |
| **История** | Undo/redo и список снимков изменений |
| **Сохранить** | Сохраняет граф, лор и настройки бота |
| **Шестерёнка** | Настройки: админ-панель, скачивание JSON, автосохранение |
| **?** | Справка по нодам с поиском |

Автосохранение можно включить в настройках. При включении редактор сохраняет сценарий примерно раз в 10 минут, но перед запуском Telegram-бота всё равно лучше нажимать **Сохранить** вручную.

### Работа со схемой

- Ноду можно добавить из левой панели или через контекстное меню правой кнопкой мыши.
- Связь создаётся перетаскиванием от выхода ноды к входу другой ноды.
- У большинства нод есть рабочие выходы слева и справа, чтобы удобнее строить крупные схемы.
- `Ctrl+C` и `Ctrl+V` копируют и вставляют выбранные ноды.
- `Delete` или `Backspace` удаляют выбранные ноды и связи.
- Группы создаются через выделение нескольких нод и команду группировки.
- Комментарии и пунктирные оранжевые связи не влияют на выполнение сценария.
- Поиск подсвечивает найденные ноды красной пунктирной рамкой.

### Лор проекта

Кнопка **Лор** открывает отдельную панель заметок. Текст лора сохраняется в JSON бота рядом с графом и подходит для:

- правил мира;
- описания персонажей;
- терминологии;
- структуры глав;
- заметок для автора или команды.

Лор не отправляется игрокам автоматически. Если часть лора должна появиться в Telegram, перенесите её в обычные сообщения или используйте плейсхолдеры/переменные.

Кнопка **Ссылка** в панели лора превращает выделенный текст в гиперссылку — в редакторе такие слова подсвечиваются синим с подчёркиванием для лучшей навигации по тексту.

### Текст и Telegram-разметка

Поля текста поддерживают плейсхолдеры вида `{{ИмяПеременной}}` и Telegram HTML-разметку. В редакторе есть панель форматирования:

| Кнопка | HTML |
|---|---|
| Жирный | `<b>текст</b>` |
| Курсив | `<i>текст</i>` |
| Подчёркивание | `<u>текст</u>` |
| Зачёркивание | `<s>текст</s>` |
| Спойлер | `<tg-spoiler>текст</tg-spoiler>` |
| Код | `<code>текст</code>` |
| Ссылка | `<a href="https://example.com">текст</a>` |

Движок отправляет текст с `parse_mode: HTML`. Не вставляйте незакрытые HTML-теги и проверяйте длинные сообщения через **Проверить**: Telegram ограничивает длину текста, caption и количество кнопок.

### Плейсхолдеры

Плейсхолдеры подставляются во время выполнения сценария.

| Плейсхолдер | Значение |
|---|---|
| `{{Имя}}` | Обычная переменная игрока |
| `{{telegram.id}}` | Telegram ID игрока |
| `{{telegram.username}}` | Username игрока |
| `{{telegram.first_name}}` | Имя из Telegram |
| `{{telegram.last_name}}` | Фамилия из Telegram |
| `{{command.args}}` | Все аргументы пользовательской команды |
| `{{command.arg1}}`, `{{command.arg2}}` | Отдельные аргументы команды |
| `{{inventory.item_key}}` | Количество конкретного предмета |
| `{{inventory.my.items}}` | Список предметов игрока |
| `{{inventory.my.amount.item_key}}` | Количество предмета в списке инвентаря |
| `{{achievement.key.title}}` | Название достижения |
| `{{achievement.key.imageUrl}}` | URL картинки достижения |
| `{{ГлобальнаяПеременная}}` | Глобальная переменная бота |

Если плейсхолдер не найден, в логах Telegram-панели появится предупреждение. Обычно причина в том, что переменная создаётся позже нужной ноды или в другой ветке.

### Все актуальные ноды

#### Точки входа и команды

| Нода | Назначение |
|---|---|
| **Глобальное меню** (`menuNode`) | Открывается командой `/start`, не сбивает текущий прогресс истории |
| **Настройки** (`settingsNode`) | Открывается командой `/settings`, подходит для языка, уведомлений и режима интерфейса |
| **Своя команда** (`customCommandNode`) | Создаёт `/profile`, `/shop`, `/map` и другие команды с псевдонимами |
| **Вызвать команду** (`invokeCommandNode`) | Открывает меню, настройки или свою команду изнутри сценария |
| **Продолжить историю** (`continueStoryNode`) | Возвращает игрока из меню/команды к последней сюжетной позиции |

У пользовательских команд доступны аргументы `{{command.args}}`, `{{command.arg1}}`, `{{command.arg2}}`. Команду можно показывать или скрывать в системном меню Telegram.

#### Сообщения игроку

| Нода | Назначение |
|---|---|
| **Сообщение** (`simpleMessageNode`) | Один текст, фото, видео, голосовое, аудио или документ |
| **Цепочка сообщений** (`messageChainNode`) | Несколько текстов и файлов подряд, у каждого элемента своя задержка |
| **Медиа-подборка** (`mediaNode`) | Несколько медиафайлов, включая Telegram-альбом для 2–10 фото/видео |
| **Ввод текста** (`textInputNode`) | Просит игрока ввести текст или число и сохраняет ответ в переменную |
| **Изменить сообщение** (`editMessageNode`) | Редактирует последнее текстовое сообщение бота; если редактировать нечего, отправляет новое |
| **Опрос или тест** (`pollNode`) | Отправляет Telegram poll или quiz с вариантами |
| **Стикер** (`stickerNode`) | Отправляет стикер по `file_id`, URL или загруженному файлу (WebP/PNG) |
| **Геолокация** (`locationNode`) | Отправляет координаты через `sendLocation`; поля подписаны: Широта (lat) и Долгота (lon) |

Для фото и видео можно включить защищённый контент. Видео можно отправлять как кружок, если файл подходит по требованиям Telegram. Стикер можно загрузить с компьютера прямо в инспекторе.

#### Логика сценария

| Нода | Назначение |
|---|---|
| **Выбор игрока** (`keyboardNode`) | Inline-кнопки Telegram, у каждой кнопки свой выход |
| **Проверка условий** (`branchingNode`) | Выбирает первую ветку, где выполнены все условия |
| **Расчёт чисел** (`formulaNode`) | Вычисляет формулу и записывает результат в числовую переменную |
| **Случайный выбор** (`randomNode`) | Генерирует число в диапазоне и выбирает подходящий интервал |
| **Задержка** (`delayNode`) | Ставит продолжение сценария в очередь на заданное время |
| **Проверка подписки** (`subscriptionCheckNode`) | Проверяет, подписан ли игрок на канал |
| **HTTP-запрос** (`httpRequestNode`) | Делает GET/POST к внешнему API и сохраняет ответ в переменную |
| **Цикл** (`loopNode`) | Выполняет тело цикла заданное число раз |
| **Выход из цикла** (`breakLoopNode`) | Досрочно завершает выбранный цикл |
| **Глобальные переменные** (`globalVariableNode`) | Меняет общие переменные, доступные всем игрокам |

В **Проверке условий** и условиях кнопок доступны источники: переменная, инвентарь, отношение, достижение и глобальная переменная. Ветка без условий работает как «иначе» и должна стоять последней.

**Формула** поддерживает арифметику (+, −, *, /, скобки), плейсхолдеры `{{переменная}}` и проценты. `50%` — это `0.5`, `выражение - 50%` уменьшает результат вдвое, `+ 25%` увеличивает на четверть. Нецелые результаты округляются вверх. В инспекторе: textarea для формулы, предпросмотр с подсветкой, мгновенный результат на основе дефолтных значений переменных, список всех доступных переменных.

**Случайный выбор** задаётся диапазоном (от/до) и набором вариантов в одну строку — название, от, до. Все интервалы должны покрывать общий диапазон без пересечений.

**HTTP-запрос** поддерживает плейсхолдеры в URL и body. Путь ответа можно указать как `data.score`, чтобы сохранить только часть JSON.

#### Прогресс, профиль и экономика

| Нода | Назначение |
|---|---|
| **Переменные** (`variableNode`) | Создаёт и меняет boolean, number и text переменные игрока |
| **Изменить инвентарь** (`inventoryNode`) | Добавляет, вычитает или устанавливает количество предметов |
| **Инвентарь** (`inventoryViewNode`) | Показывает игроку список предметов |
| **Изменить отношения** (`relationNode`) | Меняет числовые отношения с персонажем или фракцией |
| **Выдать достижение** (`achievementNode`) | Открывает достижение, сохраняет title и imageUrl |
| **Достижения** (`achievementsViewNode`) | Показывает прогресс `{{unlocked}} / {{total}}` |
| **Контрольная точка** (`checkpointNode`) | Сохраняет стабильную точку возврата |
| **Запросить промокод** (`promocodeNode`) | Просит игрока ввести промокод из админ-панели |
| **Покупка Stars** (`purchaseNode`) | Открывает оплату товара через Telegram Stars |

Инвентарь и достижения можно показывать из глобального меню или пользовательских команд, например `/profile`. Для достижений ключ должен быть стабильным: меняйте название и картинку, но не ключ, если достижение уже выдано игрокам.

#### Организация схемы

| Нода | Назначение |
|---|---|
| **Вызвать подсценарий** (`subscenarioNode`) | Переходит в переиспользуемую ветку и ждёт возврата |
| **Возврат из подсценария** (`returnNode`) | Возвращает выполнение к месту вызова |
| **Комментарий** (`commentNode`) | Авторская заметка на схеме, не выполняется ботом |
| **Группа** (`groupNode`) | Цветная область для организации больших глав и сцен |

Подсценарии удобны для магазинов, боёв, повторяемых диалогов, выдачи наград и общих экранов профиля.

### Медиафайлы

Загруженные файлы попадают в `backend/data/media/<botId>/` и раскладываются по папкам:

| Тип | Папка |
|---|---|
| Фото/изображения | `images` |
| Видео | `video` |
| Видео-кружки | `circle` |
| Голосовые | `voice` |
| Аудио | `audio` |
| Документы | `documents` |

Фронтенд/nginx раздаёт медиа напрямую через `/api/media/...`. При деплое важно не удалять `backend/data`, иначе исчезнут сценарии, токены и медиа.

### Симулятор

Симулятор запускается кнопкой **Тест** и позволяет проверить сценарий без Telegram. Он показывает:

- чат игрока;
- текущие переменные;
- инвентарь;
- отношения;
- достижения;
- глобальные переменные;
- журнал выполненных нод;
- задержки и переходы.

Симулятор поддерживает новые ноды инвентаря, достижений, ввода текста, HTTP, циклов и глобальных переменных, но реальные ограничения Telegram всё равно проверяйте через кнопку **Проверить** и тестовый запуск бота.

### Telegram-панель

Панель **Telegram** хранит токен, показывает статус и логи, запускает и останавливает бота.

| Режим | Когда используется |
|---|---|
| Long polling | Если `PUBLIC_BASE_URL` пустой |
| Webhook | Если задан публичный HTTPS `PUBLIC_BASE_URL` |

Для webhook нужен доступный извне HTTPS-домен. Для локальной разработки и внутреннего сервера без домена достаточно long polling.

Перед запуском:

1. Получите токен у [@BotFather](https://t.me/BotFather).
2. Вставьте токен в Telegram-панель.
3. Нажмите **Проверить**.
4. Сохраните бота.
5. Нажмите **Запустить**.
6. Напишите боту `/start`.

### Админ-панель

Админ-панель открывается из шестерёнки в редакторе. В ней доступны:

- список игроков;
- просмотр и редактирование переменных;
- инвентарь;
- отношения;
- достижения;
- журнал событий и выборов;
- промокоды;
- товары Telegram Stars;
- покупки;
- версии сценариев;
- резервные копии;
- аналитика и тепловая карта нод;
- рассылки и отложенные задачи.

Промокоды можно применять нодой **Запросить промокод** или командой `/promo CODE`. Товары Stars создаются в админ-панели, а в ноде **Покупка Stars** указывается ключ товара.

### Проверка сценария

Кнопка **Проверить** ищет типовые ошибки:

- незаполненные обязательные поля;
- недоступные или неподключённые выходы;
- нарушения лимитов Telegram;
- слишком длинные тексты и подписи;
- некорректные медиа;
- проблемы в кнопках и цепочках сообщений.

Цепочки сообщений дополнительно показывают предупреждения по Telegram rate limits: слишком плотная серия сообщений может упереться в лимиты API, поэтому добавляйте задержки в длинные кат-сцены и рассылки.

### Развёртывание

Минимальный запуск:

```bash
docker compose up -d --build
```

Проверка контейнеров:

```bash
docker compose ps
docker compose logs --tail=50 backend
curl -I http://localhost:4000
curl http://localhost:4001/api/bots
```

Последний запрос без авторизации должен вернуть ошибку авторизации — это нормально и означает, что API защищён.

Для продакшена с webhook:

```bash
PUBLIC_BASE_URL=https://yourdomain.com docker compose up -d --build
```

Для обновления:

```bash
git pull
docker compose up -d --build
```

Перед любым ручным деплоем сохраняйте `backend/data` и volume PostgreSQL. В `backend/data` лежат сценарии, медиа и Telegram-конфиги; в PostgreSQL — игроки, прогресс, покупки, события и промокоды.
