## 1. Быстрый старт

### Требования

- **Docker** и **Docker Compose** v2+
- Свободные порты: `4000` (фронтенд), `4001` (API)
- Доступ к интернету на сервере (для связи с Telegram API)

### Установка и запуск

```bash
git clone https://github.com/SNR93/tg-create-game-bots.git
cd tg-create-game-bots
docker compose up -d --build
```

Команда поднимает четыре контейнера:

| Контейнер | Роль | Порт (внешний) |
|---|---|---|
| `tgbot-frontend` | nginx + React-приложение | 4000 |
| `tgbot-backend` | Node.js API + движок бота | 4001 |
| `tgbot-postgres` | База данных игроков | 5433 |
| `tgbot-redis` | Хранилище сессий | 6380 |

Откройте **http://localhost:4000** и войдите под учётными данными из переменной `AUTH_USERS`.

### Управление контейнерами

```bash
docker compose down         # остановить (данные сохраняются)
docker compose up -d        # запустить снова
docker compose down -v      # остановить и удалить все данные (!)
docker compose logs -f backend   # логи бэкенда в реальном времени
```

## 2. Авторизация

Веб-интерфейс защищён страницей входа. Учётные данные задаются через переменные окружения — в репозитории их нет.

Создайте файл `.env` в корне проекта:

```env
AUTH_USERS=admin:ваш-пароль,второй:пароль
AUTH_SECRET=длинная-случайная-строка
```

- `AUTH_USERS` — пользователи в формате `логин:пароль` через запятую
- `AUTH_SECRET` — секрет подписи сессионного cookie; задайте постоянное значение, иначе после перезапуска все сессии сбрасываются

Сессия хранится **30 дней**. Для корпоративного использования добавьте отдельные учётные записи для каждого члена команды.

## 3. Интерфейс редактора

### Верхняя панель инструментов

| Элемент | Действие |
|---|---|
| **Боты** | Вернуться к списку ботов |
| Название бота | Кликните для редактирования имени; Enter — сохранить |
| **Развернуть / Свернуть** | Переключить компактный вид всех нод на холсте |
| **Лор** | Открыть заметки мира, персонажей и правил проекта |
| **Поиск** | Найти ноды по тексту, командам, ключам и настройкам |
| **Тест** | Запустить встроенный симулятор |
| **Проверить** | Проверить сценарий на ошибки и лимиты Telegram |
| **Telegram** | Запуск, остановка, статус и логи бота |
| **История** | Undo/redo и история изменений всего сценария |
| **Сохранить** / Ctrl+S | Сохранить сценарий, лор и настройки |
| **⚙** (шестерёнка) | Настройки: автосохранение, скачать JSON, открыть Админ-панель |
| **?** | Справка по нодам с поиском |

### Работа на холсте

| Действие | Способ |
|---|---|
| Переместить вид | Зажать пробел + тащить, или тащить за пустую область |
| Масштаб | Колесо мыши или кнопки Ctrl+/− |
| Выделить несколько нод | Ctrl+клик или рамкой по пустой области |
| Удалить | Выделить → Delete или Backspace |
| Копировать / Вставить | Ctrl+C / Ctrl+V |
| Отменить / Повторить | Ctrl+Z / Ctrl+Y |
| Добавить ноду | Перетащить из левой панели или через правую кнопку мыши |

### Левая панель (каталог нод)

Все доступные типы нод, сгруппированные по категориям. Перетащите ноду на холст или нажмите — она добавится в центр экрана.

### Инспектор (правая панель)

Открывается при клике на ноду. Содержит настройки: тексты, переменные, условия, медиафайлы.

Кнопка **🕐** в шапке инспектора открывает **историю изменений конкретной ноды**.

### История изменений ноды

При каждом сохранении (Ctrl+S) система фиксирует изменения данных каждой ноды: автор, время по МСК, снапшот.

| Действие | Описание |
|---|---|
| **💾 Сохранить снимок** | Ручной снапшот с произвольным комментарием |
| **▼ / ▶** | Развернуть / свернуть запись |
| **✏️ Комментарий** | Редактировать метку к версии |
| **🔍 Сравнить** | Diff по изменённым полям относительно текущего состояния |
| **↩ Восстановить** | Применить историческую версию (подтвердите через Ctrl+S) |
| **🗑 Удалить** | Удалить запись из истории |

Хранится до **50 записей** на ноду — старые обрезаются автоматически.

### Связи между нодами

- Перетащите от выхода одной ноды к входу другой для создания связи
- Большинство нод имеют выход «Продолжить» **с двух сторон** (→ правый и ← левый): оба взаимоисключающие
- Ноды с несколькими выходами (клавиатура, ветвление, случайный выбор) имеют отдельный выход на каждую ветку
- **Пунктирная оранжевая связь** — комментарий, не влияет на выполнение

### Группы

Выделите несколько нод, нажмите правую кнопку мыши → **Группа**. Группа перемещает вложенные ноды вместе и помогает визуально структурировать крупные сцены.

### Лор проекта

Кнопка **Лор** открывает текстовую панель для правил мира, описаний персонажей, терминологии и авторских заметок. Лор сохраняется в JSON бота и не отправляется игрокам. Выделенный текст можно превратить в гиперссылку — она подсвечивается синим с подчёркиванием прямо в редакторе.

### Поиск по схеме

Кнопка **Поиск** ищет по тексту нод, именам команд, ключам переменных и достижений, параметрам инспектора. Найденные ноды подсвечиваются красной пунктирной рамкой на холсте.

### Проверка сценария

Кнопка **Проверить** анализирует граф перед запуском:

- незаполненные обязательные поля
- ноды без выходов или недостижимые от старта
- нарушения лимитов Telegram (длина текста, caption, количество кнопок)
- некорректные медиафайлы
- предупреждения о rate limits в цепочках сообщений

## 4. Все типы нод

### Точки входа и команды

#### Глобальное меню (`menuNode`)
Открывается командой `/start`. Главная точка входа — показывается каждый раз при `/start`, независимо от текущей позиции в сюжете. Не сбрасывает прогресс игрока. Типичное содержимое: кнопки «Продолжить историю», «Профиль», «Магазин», «Настройки».

---

#### Настройки (`settingsNode`)
Открывается командой `/settings`. Независимая ветка для пользовательских параметров: уведомления, язык, режим отображения.

---

#### Своя команда (`customCommandNode`)
Создаёт произвольную команду Telegram: `/profile`, `/shop`, `/map` и т.д.

- **Команда** — имя без `/`, только латиница, цифры, `_`
- **Псевдонимы** — дополнительные имена через запятую
- **Описание** — текст в системном меню команд Telegram
- **Показать в меню** — отображать ли команду в меню бота

Аргументы команды доступны через `{{command.args}}`, `{{command.arg1}}`, `{{command.arg2}}` и т.д.

---

#### Вызвать команду (`invokeCommandNode`)
Открывает командную ноду (меню, настройки или свою команду) прямо из сценария — как если бы игрок написал команду сам. После нажатия «Продолжить историю» сюжет продолжается со следующего блока. Используется для автоматического показа меню между главами.

---

#### Продолжить историю (`continueStoryNode`)
Завершает независимую ветку (меню, настройки, команда) и возвращает игрока к последней сюжетной позиции. У ноды нет выхода — переход вычисляется автоматически.

---

### Сообщения

#### Сообщение (`simpleMessageNode`)
Отправляет одно сообщение: **текст**, **фото**, **видео**, **голосовое**, **аудио** или **документ**. Поддерживает плейсхолдеры, Telegram HTML-разметку, защищённый контент (спойлер, запрет пересылки) и видео-кружки (video note).

---

#### Цепочка сообщений (`messageChainNode`)
Последовательно отправляет несколько сообщений из одного блока. Каждый элемент может иметь задержку перед отправкой. Используется для диалогов, кат-сцен, серий реплик.

---

#### Медиа-подборка (`mediaNode`)
Отправляет несколько медиафайлов. Поддерживает **режим альбома** — 2–10 фото или видео одним сообщением в Telegram.

---

#### Ввод текста (`textInputNode`)
Просит игрока ввести текст или число и сохраняет ответ в переменную. Поддерживает типы `text` и `number`.

---

#### Изменить сообщение (`editMessageNode`)
Редактирует последнее текстовое сообщение бота. Если редактировать нечего, отправляет новое. Поддерживает плейсхолдеры и HTML-разметку.

---

#### Опрос или тест (`pollNode`)
Отправляет Telegram-опрос (quiz или poll) с вариантами ответов. В режиме quiz указывается правильный ответ.

---

#### Стикер (`stickerNode`)
Отправляет стикер по `file_id`, URL или загруженному файлу (WebP/PNG). Файл можно загрузить прямо в инспекторе.

---

#### Геолокация (`locationNode`)
Отправляет координаты через `sendLocation`. Поля: **Широта** (lat) и **Долгота** (lon).

---

### Логика сценария

#### Выбор игрока (`keyboardNode`)
Показывает inline-кнопки Telegram и ждёт нажатия. Каждая кнопка — отдельный выход на холсте. Подписи кнопок поддерживают плейсхолдеры.

**Условия показа кнопок:** включите «Показывать только при условиях» и задайте правила. Все условия одной кнопки проверяются как AND. Источники: переменная, инвентарь, отношение, достижение, глобальная переменная, Telegram-поле. Кнопка с условиями отмечается синей шестерёнкой в редакторе (игроку она невидима).

---

#### Проверка условий (`branchingNode`)
Проверяет значения переменных и выбирает ветку. Ветки проверяются **сверху вниз** — срабатывает первая, у которой все условия выполнены. Ветка без условий работает как «Иначе» — ставьте её последней.

**Операторы:** `==`, `!=`, `>`, `<`, `>=`, `<=`

---

#### Расчёт чисел (`formulaNode`)
Вычисляет арифметическое выражение и записывает результат в числовую переменную. Поддерживает `+`, `−`, `*`, `/`, скобки, плейсхолдеры `{{переменная}}`.

**Проценты:** `50%` = `0.5`. Запись `{{Цена}} - 20%` уменьшает значение на 20% от самого себя; `+ 10%` увеличивает. Нецелые результаты округляются вверх. Инспектор показывает мгновенный предпросмотр на основе значений по умолчанию.

---

#### Случайный выбор (`randomNode`)
Генерирует случайное целое число в заданном диапазоне и выбирает ветку, чей интервал включает это число. Интервалы должны покрывать весь диапазон без пересечений.

**Пример:** диапазон 1–10, ветка «Обычный» 1–7, ветка «Редкий» 8–10 → вероятности 70% / 30%.

---

#### Задержка (`delayNode`)
Приостанавливает сценарий на заданное время через серверную очередь. Бот продолжит автоматически — даже если игрок не пишет. Если бот был остановлен в момент ожидания, задача выполнится при следующем запуске.

---

#### Проверка подписки (`subscriptionCheckNode`)
Проверяет, подписан ли игрок на указанный канал или группу. Выходы: «Подписан» и «Не подписан».

---

#### HTTP-запрос (`httpRequestNode`)
Выполняет GET или POST к внешнему API и сохраняет ответ в переменную. URL и тело поддерживают плейсхолдеры. Путь ответа можно задать как `data.score`, чтобы извлечь нужное поле из JSON.

---

#### Цикл (`loopNode`)
Выполняет тело цикла заданное число раз. Выходы: «Итерация» (каждый шаг) и «Завершён» (после последней).

---

#### Выход из цикла (`breakLoopNode`)
Досрочно завершает выбранный активный цикл.

---

#### Глобальные переменные (`globalVariableNode`)
Изменяет переменные, общие для всех игроков бота. Операции: `set`, `increment`, `decrement`. Глобальные переменные доступны через плейсхолдеры наравне с обычными.

---

### Прогресс и профиль

#### Переменные (`variableNode`)
Создаёт и изменяет переменные игрока:

| Тип | Значения | Операции |
|---|---|---|
| `boolean` | true / false | set |
| `number` | любое число | set, increment, decrement |
| `text` | строка | set |

Переменные доступны во всём сценарии через `{{НазваниеПеременной}}`.

---

#### Изменить инвентарь (`inventoryNode`)
Добавляет, удаляет или устанавливает количество предметов в инвентаре игрока.

**Операции:** `add` (добавить), `remove` (убрать, минимум 0), `set` (установить точное значение)

---

#### Показать инвентарь (`inventoryViewNode`)
Отправляет игроку список предметов инвентаря. Заголовок и формат настраиваются в инспекторе; поддерживает плейсхолдеры.

---

#### Изменить отношения (`relationNode`)
Меняет числовой показатель отношений с персонажем или фракцией. Операции: `set`, `add`, `subtract`.

---

#### Выдать достижение (`achievementNode`)
Открывает достижение игроку. Повторный вызов не создаёт дубликат. Настройки: уникальный ключ, название, URL изображения, уведомление в чат, список переменных-наград.

> Ключ должен быть стабильным — не меняйте его после того, как достижение уже выдано игрокам.

---

#### Показать достижения (`achievementsViewNode`)
Отправляет игроку сообщение о прогрессе достижений. Шаблон по умолчанию:

```
Достижения: {{achievements.unlocked}} / {{achievements.total}}
{{achievements.list}}
```

В шаблоне доступны все плейсхолдеры достижений (см. раздел 5). Поле шаблона поддерживает HTML-форматирование.

---

#### Контрольная точка (`checkpointNode`)
Сохраняет стабильную точку возврата. При следующем `/start` игрок попадает в меню, а «Продолжить историю» возвращает его именно сюда. Используйте в начале каждой главы и после ключевых сцен.

---

### Кодекс

#### Кодекс (`codexNode`)
**Определяет** запись кодекса: уникальный ключ и текст (лор, подсказка, описание персонажа). Нода является чисто описательной — при прохождении она **не выполняется** и **не разблокирует** запись. Одна нода может содержать несколько записей.

> Не меняйте ключ после того, как запись уже была выдана игрокам.

---

#### Разблокировать кодекс (`unlockCodexNode`)
**Единственный** способ открыть или закрыть запись кодекса. Устанавливает `codex.ключ` в `true` (открыть) или `false` (закрыть). Одна нода управляет несколькими записями за раз.

При разблокировке отправляет игроку сообщение (по умолчанию «Кодекс обновлен»). Текст настраивается в инспекторе, поддерживает плейсхолдеры. При закрытии (`false`) сообщение не отправляется.

---

#### Редактировать кодекс (`editCodexNode`)
Обновляет текст существующей записи без изменения статуса блокировки. Если запись ещё не открыта — открывает её с новым текстом. Поддерживает плейсхолдеры в тексте.

---

### Монетизация и прочее

#### Запросить промокод (`promocodeNode`)
Просит игрока ввести промокод. Промокоды создаются в **Админ-панели → Промокоды**. Команда `/promo КОД` работает в любом месте сценария.

- **Текст запроса** — сообщение перед вводом
- **Текст при неверном коде** — сообщение об ошибке
- **Переменные при успехе** — список пар «переменная = значение», устанавливаются при правильном промокоде

---

#### Покупка Stars (`purchaseNode`)
Открывает оплату через Telegram Stars. Товар настраивается в **Админ-панели → Магазин Stars**; в инспекторе ноды указывается ключ товара.

---

### Структура сценария

#### Вызвать подсценарий (`subscenarioNode`)
Переходит к переиспользуемой ветке и ждёт ноды «Возврат». После возврата выполнение продолжается со следующего блока. Удобен для магазинов, боёв, повторяемых диалогов.

- **Название вызова** — метка для удобства (отображается на карточке)
- **Точка входа** — нода, с которой начинается подсценарий

---

#### Возврат из подсценария (`returnNode`)
Завершает вложенную ветку и возвращает выполнение к месту вызова. Если в схеме несколько подсценариев, можно явно указать, из какого возвращаться.

---

#### Комментарий (`commentNode`)
Авторская заметка на схеме. Не выполняется ботом. Прикрепляется пунктирной оранжевой связью к любой ноде.

---

#### Группа (`groupNode`)
Цветная область для визуальной организации нод. Создаётся через правую кнопку мыши по выделению. При перемещении группы ноды внутри двигаются вместе.

---

## 5. Переменные и плейсхолдеры

### Переменные игрока

Создаются нодой **«Переменные»**. Тип задаётся при первом создании.

| Тип | Значения | Операции |
|---|---|---|
| `boolean` | true / false | set |
| `number` | число | set, increment, decrement |
| `text` | строка | set |

### Использование плейсхолдеров

В любом текстовом поле используйте `{{ИмяПеременной}}`:

```
Привет, {{Имя}}!
У тебя {{Монеты}} монет.
Задание: {{ТекущееЗадание}}
```

Если переменная не найдена в момент выполнения, плейсхолдер остаётся как есть — и в журнале Telegram-панели появится предупреждение.

В редакторе подсказчик плейсхолдеров показывает все переменные, доступные **до текущей ноды** по пути сценария. Начните вводить `{{` — откроется выпадающий список с подходящими именами.

### Встроенные плейсхолдеры

#### Telegram-данные игрока

| Плейсхолдер | Значение |
|---|---|
| `{{telegram.id}}` | Telegram ID игрока |
| `{{telegram.username}}` | Username (без `@`) |
| `{{telegram.first_name}}` | Имя |
| `{{telegram.last_name}}` | Фамилия |

#### Аргументы команды

Доступны внутри веток `customCommandNode`:

| Плейсхолдер | Значение |
|---|---|
| `{{command.args}}` | Все аргументы после команды |
| `{{command.arg1}}` | Первый аргумент |
| `{{command.arg2}}` | Второй аргумент и т.д. |

**Пример:** игрок пишет `/give sword` → `{{command.arg1}}` = `sword`.

#### Инвентарь

| Плейсхолдер | Значение |
|---|---|
| `{{inventory.ключ}}` | Количество предмета `ключ` |
| `{{inventory.my.ключ}}` | Строка «ключ xN» |
| `{{inventory.my.amount.ключ}}` | Только число (количество) |
| `{{inventory.my.items}}` | Список всех предметов игрока |

#### Достижения

| Плейсхолдер | Значение |
|---|---|
| `{{achievement.ключ}}` | Название достижения (только если разблокировано) |
| `{{achievements.unlocked}}` | Количество полученных достижений |
| `{{achievements.total}}` | Общее количество достижений в сценарии |
| `{{achievements.list}}` | Список названий разблокированных достижений (каждое с `—`, через перенос строки) |
| `{{achievements.text.ключ}}` | Название конкретного достижения — **всегда доступно**, независимо от того, получено ли оно |

#### Кодекс

| Плейсхолдер | Значение |
|---|---|
| `{{codex.ключ}}` | Текст записи кодекса, если разблокирована; пустая строка, если нет |

#### Глобальные переменные

| Плейсхолдер | Значение |
|---|---|
| `{{ИмяГлобальнойПеременной}}` | Значение глобальной переменной (общей для всех игроков) |

## 6. Кодекс

Кодекс — система лоровых записей, которые игрок открывает по ходу сценария. Каждая запись хранит текст (история, описание персонажа, подсказка) и доступна через плейсхолдер `{{codex.ключ}}` после разблокировки.

### Как устроена работа

1. **Нода «Кодекс»** — определяет запись: задаёт ключ и текст. Размещается на схеме как справочник — через неё не обязательно проходить по сюжету.
2. **Нода «Разблокировать кодекс»** — единственное место, где запись открывается игроку. Установите значение `true` для открытия, `false` для закрытия.
3. **После разблокировки** `{{codex.ключ}}` подставляет текст записи в любое сообщение.

### Сообщение при разблокировке

В инспекторе ноды «Разблокировать кодекс» для каждой записи есть поле **«Сообщение при разблокировке»**. Если поле пустое — игроку отправляется «Кодекс обновлен». Текст поддерживает плейсхолдеры.

### Обновление текста записи

Нода **«Редактировать кодекс»** обновляет текст уже открытой записи, не меняя статус блокировки. Если запись не открыта — открывает её с новым текстом.

### Пример использования

```
1. Нода «Кодекс»: ключ = harback, текст = «Харбэк Крепкоплечий — легендарный гном-кузнец...»
2. Позже по сюжету — Нода «Разблокировать кодекс»: codex.harback = true
3. В сообщении: «Вы узнали кое-что о Харбэке: {{codex.harback}}»
```

1. Откройте [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot`
3. Введите имя и username (должен заканчиваться на `bot`)

## 9. Админ-панель

Открывается через ⚙ → **Админ-панель** или из настроек редактора.

### Игроки

Список всех пользователей, запустивших бота. Поиск по имени, username или Telegram ID.

Для каждого игрока доступно:
- Просмотр и редактирование переменных (тип и значение)
- Управление инвентарём (добавить, удалить, изменить количество)
- Просмотр и изменение отношений с персонажами
- Управление достижениями (добавить, удалить)
- Журнал выборов (какие кнопки нажимал и когда)
- Сброс прогресса (с возможностью сохранить отдельные переменные)
- Полное удаление игрока

### Аналитика

- **Воронка конверсии** — от старта до завершения сценария
- **Тепловая карта нод** — горизонтальный бар-чарт по количеству посещений
- **Популярные выборы** — какие кнопки нажимают чаще всего
- **Сводка** — количество оплат, заработанные Stars, реферальные игроки

### Промокоды

Создание и управление промокодами. Награды задаются в формате JSON:

```json
{
  "variables": { "Монеты": 100, "VIP": true },
  "inventory": { "меч_рыцаря": 1 }
}
```

- **Лимит использований** — оставьте пустым для неограниченного; промокод деактивируется автоматически
- Промокоды хранятся в верхнем регистре

### Магазин Stars

Товары для покупки через Telegram Stars:
- **Ключ** — стабильный идентификатор (используется в ноде «Покупка Stars»)
- **Название** и **описание** — отображаются в инвойсе Telegram
- **Цена** — в единицах Telegram Stars (минимум 1)
- **Награды** — JSON с переменными и предметами, выдаются после успешной оплаты

### Версии сценария

Снимки состояния сценария с возможностью публикации. Подробнее — в разделе 10.

### Резервные копии

Ручные и Telegram-бэкапы. Подробнее — в разделе 10.

### Рассылки

Отложенные сообщения всем игрокам. Подробнее — в разделе 13.

### Роли

Управление доступом к проекту: `owner`, `editor`, `viewer`.

## 10. Версии и резервные копии

### Версии сценария

Снимок фиксирует текущее состояние сценария в базе данных.

**Создать**: Админ-панель → Версии → «Создать снимок версии»

После создания снимок можно **опубликовать** с указанием процента выката (rollout):
- `100%` — все новые игроки получают эту версию
- `10%` — 10% новых игроков, остальные остаются на предыдущей

Уже начавшим игрокам версия не меняется. Хранится до **50 архивных версий** на бота — при создании новой самые старые архивные удаляются автоматически. Опубликованные версии не затрагиваются.

### Резервные копии (файловые)

**Создать**: Админ-панель → Резервные копии → «Создать резервную копию»

**Восстановить**: нажмите «Восстановить» рядом с нужной копией. Сценарий в редакторе будет заменён. Обновите страницу после восстановления.

### Telegram-бэкап

Создаёт `.tar.gz` архив и отправляет его в Telegram-чат.

Архив содержит:
- сценарии всех ботов (`backend/data/bots/`)
- медиафайлы (`backend/data/media/`)
- Telegram-конфиги (`backend/data/telegram/`)
- дамп PostgreSQL `db/tgbot.sql` (игроки, переменные, прогресс, покупки)

**Восстановление из Telegram-бэкапа:**

```bash
# Распаковать архив
tar -xzf backup.tar.gz

# Восстановить файлы
cp -r data/* /путь/к/backend/data/

# Восстановить базу данных
docker exec -i tgbot-postgres psql -U tgbot tgbot < db/tgbot.sql
```

## 14. Развёртывание на сервере

### Требования

- Ubuntu 20.04+ / Debian 11+ или любой Linux с Docker
- Docker Engine 24+ и Docker Compose v2+
- Минимум 1 ГБ RAM, 10 ГБ диска
- Публичный HTTPS-домен для webhook-режима (рекомендуется для продакшена)

### Установка Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

### Запуск

```bash
git clone https://github.com/SNR93/tg-create-game-bots.git
cd tg-create-game-bots
cp .env.example .env   # создайте файл .env и задайте AUTH_USERS, AUTH_SECRET
docker compose up -d --build
```

### Webhook-режим (для продакшена)

Добавьте `PUBLIC_BASE_URL` в `docker-compose.yml` или `.env`:

```env
PUBLIC_BASE_URL=https://yourdomain.com
```

Или передайте при запуске:

```bash
PUBLIC_BASE_URL=https://yourdomain.com docker compose up -d
```

### Обратный прокси (nginx)

```nginx
server {
    listen 443 ssl;
    server_name yourdomain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://localhost:4000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### Обновление

```bash
git pull
docker compose up -d --build
```

### Расположение данных

| Данные | Путь |
|---|---|
| Сценарии ботов | `./backend/data/bots/` |
| Медиафайлы | `./backend/data/media/` |
| Telegram-конфиги (токены) | `./backend/data/telegram/` |
| Игровые данные | PostgreSQL volume `postgres-data` |

> **Никогда не удаляйте** `backend/data` при обновлении. В нём хранятся сценарии, медиа и Telegram-конфиги. Данные игроков находятся в PostgreSQL volume и сохраняются при `docker compose down`.

### Ручной бэкап базы данных

```bash
# Создать дамп
docker exec tgbot-postgres pg_dump -U tgbot tgbot > backup.sql

# Восстановить
docker exec -i tgbot-postgres psql -U tgbot tgbot < backup.sql
```

## 15. Часто задаваемые вопросы

**Бот не запускается: «Telegram отклонил токен»**
Проверьте токен у @BotFather. Убедитесь, что бот не удалён и токен не был сброшен командой `/revoke`.

**«Этот бот уже запущен в другом месте»**
Токен используется другим процессом. Остановите другой экземпляр или подождите несколько секунд — Telegram сам разрывает старое соединение.

**Плейсхолдер `{{Переменная}}` виден в сообщении как есть**
Переменная не найдена в момент выполнения этой ноды. Проверьте: нода «Переменные» стоит **до** ноды с текстом по пути сценария; имя совпадает с учётом регистра. В журнале Telegram-панели появится предупреждение с точным именем.

**`{{codex.ключ}}` возвращает пустую строку**
Запись не была разблокирована для этого игрока. Убедитесь, что нода «Разблокировать кодекс» с этим ключом и значением `true` стоит раньше по пути сценария.

**После восстановления из бэкапа ничего не изменилось**
Обновите страницу (Ctrl+Shift+R) после восстановления.

**Как переименовать переменную?**
Прямого переименования нет. Создайте новую ноду «Переменные» с новым именем и замените все плейсхолдеры в текстах. Старую переменную можно удалить через Админ-панель.

**Сколько игроков поддерживается?**
Без встроенных ограничений. Узкое место — Telegram API (30 сообщений/сек глобально), автоматически регулируется встроенным rate limiter'ом.

**Как запустить несколько ботов?**
Создайте любое количество ботов через **+ Новый бот**. Каждый использует отдельный Telegram-токен и работает независимо — все обслуживаются одним бэкендом.

**Как посмотреть логи контейнеров?**

```bash
docker logs tgbot-backend -f     # бэкенд в реальном времени
docker logs tgbot-frontend       # nginx
docker logs tgbot-postgres       # база данных
```
