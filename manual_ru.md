## 1. Быстрый старт

### Требования

- **Docker** и **Docker Compose** (v2+)
- Свободные порты: `4000` (фронтенд), `4001` (API)
- Интернет-соединение на сервере (для связи с Telegram API)

### Установка

```bash
git clone https://github.com/SNR93/tg-create-game-bots.git
cd tg-create-game-bots
docker compose up -d --build
```

Команда поднимает 4 контейнера:

| Контейнер | Роль | Порт |
|---|---|---|
| `tgbot-frontend` | nginx + React-приложение | 4000 |
| `tgbot-backend` | Node.js API + движок бота | 4001 |
| `tgbot-postgres` | База данных игроков | 5433 (локальный) |
| `tgbot-redis` | Хранилище сессий | 6380 (локальный) |

### Первый запуск

1. Откройте **http://localhost:4000**
2. Нажмите **+ Новый бот**
3. Введите название и нажмите **Создать**
4. Откроется редактор с нодой «Глобальное меню» — с неё начинается любой сценарий

### Остановка и перезапуск

```bash
docker compose down       # остановить (данные сохраняются)
docker compose up -d      # запустить снова
docker compose down -v    # остановить и удалить все данные (!)
```

## 2. Интерфейс редактора

### Панель инструментов (верхняя)

| Кнопка | Действие |
|---|---|
| Название бота | Редактировать имя (Enter для сохранения) |
| **Сохранить** / Ctrl+S | Сохранить сценарий |
| **Скачать JSON** | Выгрузить сценарий как файл |
| **Тест** | Открыть встроенный симулятор |
| **Telegram** | Подключить и запустить бота |
| **Админ** | Открыть панель управления игроками |
| **История** | Просмотр и откат изменений (undo/redo) |
| **?** | Справка по нодам |
| ← | Вернуться к списку ботов |

### Холст

- **Перемещение**: зажать пробел + тащить, или тащить за пустую область
- **Масштаб**: колесо мыши или Ctrl+/−
- **Выделить несколько нод**: зажать Ctrl и кликать, или тащить рамкой
- **Удалить ноду**: выделить → Del или Backspace
- **Дублировать**: выделить → Ctrl+D

### Панель нод (левая)

Каталог всех доступных нод, разбитых по категориям. Перетащите ноду на холст или нажмите на неё — она добавится в центр экрана.

### Инспектор (правая панель)

Появляется при клике на ноду. Настройки ноды: тексты, переменные, условия, файлы.

### Связи между нодами

- Перетащите от **правого выхода** одной ноды к **левому входу** другой
- Большинство нод имеют один выход «Продолжить»
- Клавиатуры и ветвления имеют **отдельный выход на каждый вариант**
- Связи-комментарии (пунктирные оранжевые) не влияют на выполнение

### Группы

Выделите несколько нод (Ctrl+клик или рамкой), нажмите правую кнопку → **Группа**. Группа объединяет ноды визуально и перемещает их вместе.

## 3. Все типы нод

### Точки входа и команды

#### Глобальное меню (`menuNode`)
**Открывается командой `/start`**. Это главная точка входа в бота — показывается каждый раз, когда игрок пишет /start, независимо от текущего места в истории.

Типичное содержимое: кнопки «Продолжить историю», «Профиль», «Магазин», «Настройки».

> Нода не перезаписывает прогресс игрока. После нажатия любой кнопки управление переходит к подключённой ноде.

---

#### Настройки (`settingsNode`)
**Открывается командой `/settings`**. Независимая ветка для пользовательских параметров: уведомления, язык, режим отображения.

---

#### Своя команда (`customCommandNode`)
Создаёт произвольную команду Telegram (например, `/profile`, `/shop`, `/map`).

**Настройки в инспекторе:**
- **Команда** — имя без `/`, только латинские буквы, цифры, `_`
- **Псевдонимы** — дополнительные имена через запятую
- **Описание** — текст в меню команд Telegram
- **Показать в меню** — отображать ли команду в системном меню Telegram

Аргументы команды доступны через плейсхолдеры `{{command.args}}`, `{{command.arg1}}`, `{{command.arg2}}` и т.д.

---

#### Вызвать команду (`invokeCommandNode`)
Принудительно открывает командную ноду прямо из сценария — как если бы игрок сам написал команду.

**Использование:** выберите целевую ноду (меню, настройки или свою команду) в выпадающем списке. Выбранная нода откроется в режиме оверлея. После нажатия «Продолжить историю» сюжет продолжится со следующего блока после этой ноды.

**Пример:** конец главы → автоматически показать меню → следующая глава.

---

#### Продолжить историю (`continueStoryNode`)
Завершает независимую ветку (меню, настройки, команда) и возвращает игрока к последней позиции в истории.

> У ноды нет выхода — переход вычисляется автоматически.

---

### Сообщения

#### Сообщение (`simpleMessageNode`)
Отправляет одно сообщение: **текст**, **фото**, **видео**, **голосовое**, **аудио** или **документ**.

**Поддерживает:**
- Плейсхолдеры `{{ИмяПеременной}}` в тексте
- Защищённый контент (спойлер для фото/видео, запрет пересылки)
- Видео-кружки (video note)

---

#### Цепочка сообщений (`messageChainNode`)
Последовательно отправляет несколько сообщений из одного блока. Каждое сообщение может содержать задержку перед отправкой.

**Когда использовать:** диалоги, кат-сцены, серии реплик с единым переходом после.

---

#### Медиа-подборка (`mediaNode`)
Отправляет один или несколько медиафайлов. Поддерживает **режим альбома** (2–10 фото или видео одним сообщением).

---

### Логика сценария

#### Выбор игрока (`keyboardNode`)
Показывает inline-кнопки Telegram и ждёт нажатия. Каждая кнопка — отдельный выход на холсте.

**Настройки:** добавляйте и удаляйте кнопки, редактируйте подписи (поддерживают плейсхолдеры).

---

#### Проверка условий (`branchingNode`)
Проверяет значения переменных и выбирает подходящую ветку. Ветки проверяются **сверху вниз**, срабатывает первая, у которой выполнены все условия.

**Ветка без условий** работает как «Иначе» — поставьте её последней.

**Операторы:** `==`, `!=`, `>`, `<`, `>=`, `<=`

---

#### Случайный выбор (`randomNode`)
Выбирает одну из веток случайно с учётом весов. Вес — относительная вероятность (не процент).

**Пример:** вариант A с весом 8, вариант B с весом 2 → A выбирается в 80% случаев.

---

#### Расчёт чисел (`formulaNode`)
Выполняет арифметику с числовыми переменными: `=`, `+`, `−`, `×`, `÷`.

---

#### Задержка (`delayNode`)
Приостанавливает сценарий на заданное количество секунд через серверную очередь. Бот продолжит сценарий автоматически — даже если игрок не пишет.

---

### Прогресс и экономика

#### Переменные (`variableNode`)
Создаёт и изменяет переменные:

| Тип | Значения | Операции |
|---|---|---|
| `boolean` | true / false | set |
| `number` | любое число | set, increment, decrement |
| `text` | строка | set |

Переменные доступны во всём сценарии через плейсхолдеры `{{НазваниеПеременной}}`.

---

#### Изменить инвентарь (`inventoryNode`)
Добавляет, удаляет или устанавливает количество предметов в инвентаре игрока.

**Операции:** `add` (добавить), `remove` (удалить, минимум 0), `set` (установить)

---

#### Изменить отношения (`relationNode`)
Меняет числовой показатель отношений с персонажем или фракцией.

**Операции:** `set`, `add`, `subtract`

---

#### Выдать достижение (`achievementNode`)
Открывает достижение игроку. Повторный вызов не создаёт дубликат. Можно показать уведомление в чате.

---

#### Контрольная точка (`checkpointNode`)
Сохраняет стабильную точку возврата. Если игрок снова напишет `/start` → перейдёт к меню, а «Продолжить историю» вернёт его именно сюда.

> Используйте в начале каждой главы или после важных сцен.

---

#### Запросить промокод (`promocodeNode`)
Запрашивает у игрока ввод промокода. Промокоды создаются в **Админ-панели → Промокоды**. Команда `/promo КОД` также работает в любом месте сценария.

---

#### Покупка Stars (`purchaseNode`)
Открывает оплату через Telegram Stars. Товар настраивается в **Админ-панели → Магазин Stars**.

---

### Структура

#### Вызвать подсценарий (`subscenarioNode`)
Переходит к переиспользуемой ветке и после ноды «Возврат» продолжает выполнение со следующего блока.

**Использование:** выберите точку входа подсценария в инспекторе.

---

#### Возврат из подсценария (`returnNode`)
Завершает вложенную ветку и возвращает выполнение к месту вызова.

---

#### Комментарий (`commentNode`)
Заметка автора на схеме. Не влияет на выполнение бота. Можно прикрепить пунктирной связью к любой ноде.

---

#### Группа (`groupNode`)
Цветная область для организации нод. Создаётся правой кнопкой по выделению. При перемещении группы ноды внутри двигаются вместе.

---

## 4. Переменные и плейсхолдеры

### Создание переменных

Переменные создаются нодой **«Переменные»**. Тип определяется при первом создании и должен оставаться согласованным.

### Использование в тексте

В любом текстовом поле используйте двойные фигурные скобки:

```
Привет, {{Имя}}!
У тебя {{Монеты}} монет.
Отношения с Алисой: {{Доверие_Алисы}}
```

Если переменная не найдена, плейсхолдер остаётся видимым (`{{Имя}}`) — и сразу в логах бота появится предупреждение, чтобы вы могли быстро найти ошибку.

### Встроенные переменные команд

Внутри ветки `customCommandNode` доступны:

| Плейсхолдер | Значение |
|---|---|
| `{{command.args}}` | Все аргументы после команды |
| `{{command.arg1}}` | Первый аргумент |
| `{{command.arg2}}` | Второй аргумент |
| … | … |

**Пример:** игрок пишет `/give sword` → `{{command.arg1}}` = `sword`

## 5. Подключение Telegram-бота

### Шаг 1: Создать бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot`
3. Введите имя и username (должен заканчиваться на `bot`)
4. Скопируйте токен вида `1234567890:ABCdef...`

### Шаг 2: Подключить токен в редакторе

1. Откройте редактор бота
2. Нажмите кнопку **Telegram** в верхней панели
3. Вставьте токен и нажмите **Запустить**

Бот начнёт работать в режиме **long polling** — он самостоятельно опрашивает Telegram на наличие сообщений.

### Режимы работы

| Режим | Когда используется |
|---|---|
| **Long polling** | Без `PUBLIC_BASE_URL` — для разработки и локальных машин |
| **Webhook** | При установленном `PUBLIC_BASE_URL` — для продакшена |

### Просмотр логов

В панели Telegram отображаются последние 300 событий: входящие сообщения, выборы игроков, ошибки, предупреждения о несуществующих переменных.

### Остановка бота

Нажмите **Остановить** в панели Telegram. Бот можно запустить снова в любой момент.

## 6. Тестирование в симуляторе

Симулятор позволяет протестировать сценарий прямо в браузере без подключения к Telegram.

### Открытие

Нажмите кнопку **Тест** в верхней панели редактора.

### Интерфейс

| Панель | Содержимое |
|---|---|
| **Левая** | Текущие значения переменных (можно редактировать вручную) |
| **Центральная** | Имитация чата Telegram |
| **Правая** | Журнал выполнения с указанием нод |

### Команды бота

Нажмите кнопку ☰ в поле ввода, чтобы запустить любую команду из тех, что настроены на схеме: `/start`, `/settings`, свои команды.

### Особенности симулятора

- **Задержки** отображаются таймером с кнопкой «Пропустить»
- **Покупка Stars** показывает карточку с кнопками «Оплатить (симуляция)» и «Пропустить»
- **Промокод** запрашивает ввод, но не проверяет реальные промокоды — любой текст принимается
- Переменные можно менять прямо во время выполнения для проверки разных ветвлений

## 7. Админ-панель

Открывается кнопкой **Админ** в верхней панели редактора.

### Игроки

Список всех пользователей, которые запустили бота. Поиск по имени, username или Telegram ID.

**Для каждого игрока доступно:**
- Редактирование переменных (тип и значение)
- Управление инвентарём (добавить, удалить, изменить количество)
- Просмотр и изменение отношений с персонажами
- Управление достижениями
- Журнал выборов (какие кнопки нажимал)
- Сброс прогресса
- Полное удаление

### Аналитика

- **Воронка конверсии** — от «начали игру» до «прошли сценарий»
- **Топ посещаемых нод** — горизонтальный бар-чарт по количеству визитов
- **Популярные выборы** — какие кнопки нажимают чаще всего
- **Сводка** — количество оплат, Stars заработано, реферальные игроки

### Промокоды

Создание промокодов с наградами. Награды указываются в формате JSON:

```json
{
  "variables": { "Монеты": 100 },
  "inventory": { "ключ_подвала": 1 }
}
```

**Параметры:**
- **Лимит использований** — оставьте пустым для неограниченного
- Промокод деактивируется автоматически после исчерпания лимита

### Магазин Stars

Добавление товаров для покупки через Telegram Stars:
- **Ключ** — стабильный идентификатор товара (используется в ноде «Покупка»)
- **Название** и **описание** — показываются в инвойсе Telegram
- **Цена** — в единицах Telegram Stars (минимум 1)
- **Награды** — JSON с переменными/предметами, которые выдаются после оплаты

### Роли

Управление доступом к проекту: `owner`, `editor`, `viewer`.

> Примечание: роли хранятся в базе данных, но принудительная проверка на уровне API пока не реализована — планируется в следующих версиях.

## 8. Версии и резервные копии

### Версии сценария

**Создать снимок**: Adminпанель → Версии → «Создать снимок версии»

Снимок фиксирует текущее состояние сценария. После создания его можно **опубликовать** с указанием **процента выката** (rollout):

- `100%` — все новые игроки получают эту версию
- `10%` — 10% новых игроков получают новую версию, остальные — старую

Уже начавшие игрокам назначенная версия не меняется.

### Резервные копии

**Создать копию**: Adminпанель → Резервные копии → «Создать резервную копию»

**Восстановить**: нажмите «Восстановить» рядом с нужной копией. Текущий сценарий в редакторе будет заменён. Обновите страницу после восстановления.

## 9. Монетизация через Telegram Stars

### Настройка

1. В **Админ-панели → Магазин Stars** создайте товар
2. Укажите ключ, название, цену в Stars, награды
3. В редакторе добавьте ноду **«Покупка Stars»**
4. В инспекторе укажите ключ товара

### Процесс оплаты

1. Бот отправляет инвойс Telegram
2. Игрок нажимает «Оплатить»
3. Telegram подтверждает платёж
4. Бот выдаёт награды и продолжает сценарий

> Telegram Stars не требуют подключения платёжного провайдера — всё обрабатывается внутри Telegram.

---

## 10. Промокоды

### Создание в Админ-панели

1. Откройте **Админ-панель → Промокоды**
2. Введите код (автоматически переводится в верхний регистр)
3. Укажите награды в JSON
4. Задайте лимит использований (необязательно)
5. Нажмите **Добавить**

### Использование игроком

**Способ 1:** через ноду **«Запросить промокод»** в сценарии  
**Способ 2:** игрок пишет `/promo КОД` в любом месте

---

## 11. Рассылки и отложенные задачи

### Запланировать рассылку

1. **Админ-панель → Очередь** → «Запланировать рассылку»
2. Введите текст (поддерживает лимиты Telegram)
3. Укажите дату и время
4. Нажмите **Поставить в очередь**

Рассылка будет отправлена всем игрокам, у которых есть chat_id (т.е. запустившим бота).

### Отложенное продолжение

Нода **«Задержка»** ставит продолжение сценария в серверную очередь. Очередь проверяется каждые 2 секунды. Если бот остановлен в момент выполнения задачи — она выполнится при следующем запуске.

## 12. Развёртывание на сервере

### Требования к серверу

- Ubuntu 20.04+ / Debian 11+ (или любой Linux с Docker)
- Docker Engine 24+ и Docker Compose v2+
- Минимум 1 ГБ RAM, 10 ГБ диска
- Домен с SSL для webhook-режима (рекомендуется)

### Установка Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

### Клонирование и запуск

```bash
git clone https://github.com/SNR93/tg-create-game-bots.git
cd tg-create-game-bots
docker compose up -d --build
```

### Настройка обратного прокси (nginx + SSL)

Пример конфигурации для Nginx Proxy Manager или ручного nginx:

```nginx
server {
    listen 443 ssl;
    server_name yourdomain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://localhost:4000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### Включение webhook-режима

Добавьте `PUBLIC_BASE_URL` в `docker-compose.yml`:

```yaml
backend:
  environment:
    - PUBLIC_BASE_URL=https://yourdomain.com
```

Или передайте при запуске:

```bash
PUBLIC_BASE_URL=https://yourdomain.com docker compose up -d
```

В webhook-режиме бот не опрашивает Telegram — Telegram сам шлёт обновления на сервер. Это быстрее и надёжнее для продакшена.

### Резервное копирование данных

Все данные хранятся в двух местах:

| Данные | Расположение |
|---|---|
| Сценарии ботов | `./backend/data/bots/` |
| Медиафайлы | `./backend/data/media/` |
| Конфигурация Telegram | `./backend/data/telegram/` |
| Игровые данные (игроки, переменные) | PostgreSQL volume `postgres-data` |

**Бэкап базы данных:**

```bash
docker exec tgbot-postgres pg_dump -U tgbot tgbot > backup.sql
```

**Восстановление:**

```bash
docker exec -i tgbot-postgres psql -U tgbot tgbot < backup.sql
```

### Обновление

```bash
git pull
docker compose up -d --build
```

## 13. Часто задаваемые вопросы

### Бот не запускается, ошибка «Telegram отклонил токен»
Проверьте токен в настройках бота у @BotFather. Убедитесь, что бот не был удалён и токен не был сброшен.

### «Этот Telegram-бот уже запущен в другом месте»
Токен используется в другом процессе или экземпляре приложения. Остановите другой экземпляр или подождите несколько секунд — Telegram сам разрывает старое соединение.

### Не удалось подключиться к Telegram API
Сервер не имеет доступа к `api.telegram.org`. Проверьте интернет-соединение и файервол. В некоторых странах Telegram заблокирован — используйте VPN на сервере.

### Плейсхолдер `{{Переменная}}` отображается как есть в сообщении
Переменная не найдена в момент выполнения этой ноды. Проверьте:
- Нода **«Переменные»** для этой переменной стоит **до** ноды с текстом в сценарии
- Имя переменной совпадает (учитывайте регистр)
- В логах Telegram-панели будет предупреждение с точным именем отсутствующей переменной

### После восстановления из резервной копии ничего не изменилось
Обновите страницу браузера (Ctrl+Shift+R) после восстановления.

### Как переименовать существующую переменную?
Прямого переименования нет. Создайте новую ноду «Переменные» с новым именем, установите нужное значение, и замените все плейсхолдеры в текстах.

### Сколько игроков поддерживается?
Ограничений на количество игроков нет. Узкое место — API Telegram (30 сообщений/сек глобально), которое автоматически регулируется встроенным rate limiter'ом.

### Как запустить несколько ботов?
Создайте любое количество ботов через **+ Новый бот**. Каждый бот работает с отдельным Telegram-токеном и независимо. Все они обслуживаются одним бэкендом.

### Как посмотреть логи контейнеров?

```bash
docker logs tgbot-backend -f    # логи бэкенда в реальном времени
docker logs tgbot-frontend      # логи nginx
docker logs tgbot-postgres      # логи базы данных
```
